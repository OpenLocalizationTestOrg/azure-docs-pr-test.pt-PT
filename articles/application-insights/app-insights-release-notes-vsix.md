## <a name="version-70"></a>Versão 7.0
### <a name="azure-application-insights-trends"></a>Azure Application Insights Trends
As informações de aplicação do Azure é uma ferramenta de novo no Visual Studio que pode utilizar para o ajudar a analisar a forma como a sua aplicação funciona ao longo do tempo. Para começar a utilizar, no **Application Insights** barra de ferramentas no botão ou na janela Pesquisa do Application Insights, escolha **explorar tendências de telemetria**. Ou, no **vista** menu, clique em **outras janelas**e, em seguida, clique em **Application Insights Trends**. Escolha uma das cinco consultas comuns para começar. Pode analisar os diferentes conjuntos de dados com base nos tipos de telemetria, intervalos de tempo e outras propriedades. Para localizar anomalias nos seus dados, escolha uma das opções anómalas do **tipo de vista** na lista pendente. As opções de filtragem na parte inferior da janela facilitam a maximização de subconjuntos específicos da sua telemetria.

![O Application Insights Trends](./media/app-insights-release-notes-vsix/Trends.png)

### <a name="exceptions-in-codelens"></a>Exceções no CodeLens
Telemetria de exceção é apresentada no CodeLens. Se tiver ligado seu projeto para o serviço do Application Insights, verá o número de exceções que ocorreram cada método na produção nas últimas 24 horas. De CodeLens, pode saltar para pesquisa ou tendências para investigar as exceções em mais detalhe.

![Exceções no CodeLens](./media/app-insights-release-notes-vsix/ExceptionsCodeLens.png)

### <a name="aspnet-core-support"></a>Suporte de ASP.NET Core
O Application Insights suporta agora o ASP.NET Core RC2 projetos no Visual Studio. Pode adicionar o Application Insights para novos projetos de ASP.NET Core RC2 do **novo projeto** caixa de diálogo, como no seguinte captura de ecrã. Em alternativa, pode adicioná-lo a um projeto existente, clique com o botão direito no projeto no Explorador de soluções e, em seguida, clique em **adicionar telemetria do Application Insights**.

![Suporte de ASP.NET Core](./media/app-insights-release-notes-vsix/NetCoreSupport.png)

ASP.NET 5 RC1 e projetos de ASP.NET Core RC2 também tem novo suporte na janela ferramentas de diagnóstico. Irá ver eventos de Application Insights, como pedidos e exceções da sua aplicação ASP.NET, enquanto a depuração localmente no seu PC. Cada evento, clique em **pesquisa** para desagregar para obter mais informações.

![Suportem de ferramentas de diagnóstico](./media/app-insights-release-notes-vsix/DiagnosticTools.png)

### <a name="hockeyapp-for-universal-windows-apps"></a>HockeyApp para aplicações do Universal Windows
Para além de comentários beta do utilizador e de distribuição, o HockeyApp fornece falhas symbolicated Reporting Services para as suas aplicações Universal do Windows. Que fizemos ainda mais fáceis de adicionar o SDK do HockeyApp: com o botão direito no projeto Universal do Windows e, em seguida, clique em **Hockey aplicação – ativar análise de falhas**. Isto instala o SDK, configura a recolha de falhas e aprovisiona um recurso do HockeyApp na nuvem, tudo sem carregar a aplicação para o serviço do HockeyApp.

Outras novas funcionalidades:

* Fizemos a experiência de pesquisa do Application Insights mais rápida e mais intuitiva. Agora, intervalos de tempo e os filtros de detalhe são automaticamente aplicados à medida que seleciona-los.
* Também na pesquisa do Application Insights, agora há uma opção para ir para o código diretamente a partir da telemetria do pedido.
* Fizemos melhoramentos à experiência de início de sessão HockeyApp.
* As ferramentas de diagnóstico, informações de telemetria de produção para exceções são apresentadas.

## <a name="version-52"></a>Versão 5.2
Estamos satisfeitos anunciar a introdução dos cenários do HockeyApp no Visual Studio. A integração de primeira é a distribuição beta de aplicações do Universal Windows e aplicações Windows Forms no Visual Studio.

Com a distribuição beta, carregue antecipadas versões das suas aplicações para o HockeyApp para distribuição a um subconjunto selecionado de clientes ou testers. Distribuição beta, combinada com HockeyApp falhas coleção e o utilizador comentários funcionalidades, pode fornecer informações importantes sobre a sua aplicação antes de efetuar uma versão abrangente. Pode utilizar estas informações para resolver problemas com a sua aplicação para que possa evitar ou minimizar os problemas futuros, tais como classificações de baixa de aplicação, comentários negativos e assim sucessivamente.

Veja como simples é carregar baseia-se para a distribuição beta no Visual Studio.

### <a name="universal-windows-apps"></a>Aplicações universal do Windows
O menu de contexto para um nó de projeto de aplicação Universal do Windows inclui agora uma opção para carregar a compilação para o HockeyApp.

![Menu de contexto do projeto para aplicações do Universal Windows](./media/app-insights-release-notes-vsix/UniversalContextMenu.png)

Escolha o item e o HockeyApp é aberta a caixa de diálogo de carregamento. Vai precisar de uma conta do HockeyApp para carregar a compilação. Se for um novo utilizador, não se preocupe. Criar uma conta é um processo simple.

Quando estiver ligado, verá o formulário de carregamento na caixa de diálogo.

![Carregar a caixa de diálogo para aplicações do Universal Windows](./media/app-insights-release-notes-vsix/UniversalUploadDialog.png)

Selecione o conteúdo a carregar (um ficheiro. appxbundle ou. AppX) e, em seguida, escolha as opções de versão no assistente. Opcionalmente, pode adicionar notas de versão na página seguinte. Escolha **concluir** para iniciar o carregamento.

Quando o carregamento estiver concluído, é apresentada uma notificação de HockeyApp com a confirmação e uma hiperligação para a aplicação no portal do HockeyApp.

![Carregar notificação concluída](./media/app-insights-release-notes-vsix/UploadComplete.png)

Já está! Que acabou de carregar uma compilação para distribuição beta com apenas alguns cliques.

Pode gerir a aplicação de várias formas no portal do HockeyApp. Isto inclui inviting utilizadores, visualizar relatórios de falhas e comentários, alterar os detalhes e assim sucessivamente.

![Portal do HockeyApp](./media/app-insights-release-notes-vsix/HockeyAppPortal.png)

Consulte o [Base de dados de conhecimento do HockeyApp](http://support.hockeyapp.net/kb/app-management-2) para obter mais detalhes sobre a gestão de aplicações.

### <a name="windows-forms-apps"></a>Aplicações de formulários do Windows
O menu de contexto para um nó do projeto Windows formulário inclui agora uma opção para carregar a compilação para o HockeyApp.

![Menu de contexto do projeto para aplicações de Windows Forms](./media/app-insights-release-notes-vsix/WinFormContextMenu.png)

Esta ação abre o diálogo de carregamento do HockeyApp, que é semelhante de uma aplicação Universal do Windows.

![Carregar a caixa de diálogo para aplicações de Windows Forms](./media/app-insights-release-notes-vsix/WinFormsUploadDialog.png)

Tenha em atenção um novo campo neste assistente, para especificar a versão da aplicação. Para aplicações Universal do Windows, as informações são povoadas do manifesto. Aplicações do Windows Forms, Infelizmente, não tem um equivalente para esta funcionalidade. Terá de especificá-los manualmente.

O resto do fluxo é semelhante às aplicações Universal do Windows: escolha compilação e opções das versões, adicione versão notas, carregar e gerir no portal do HockeyApp.

É tão simple como que. Experimentá-la e informe-na sua opinião.

## <a name="version-43"></a>Versão 4.3
### <a name="search-telemetry-from-local-debug-sessions"></a>Telemetria de pesquisa de sessões de depuração local
Com esta versão, agora pode pesquisar de telemetria do Application Insights gerada na sessão de depuração do Visual Studio. Antes, pode utilizar a pesquisa apenas se registou a aplicação com o Application Insights. Agora, a aplicação só tem de ter o Application Insights SDK instalado para procurar a telemetria local.

Se tiver uma aplicação ASP.NET com o Application Insights SDK, siga os passos seguintes para utilizar a procura.

1. Depurar a aplicação.
2. Abra pesquisa do Application Insights de uma das seguintes formas:
   * No **vista** menu, clique em **outras janelas**e, em seguida, clique em **pesquisa do Application Insights**.
   * Clique em de **Application Insights** botão da barra de ferramentas.
   * No Explorador de soluções, expanda **Applicationinsights**e, em seguida, clique em **telemetria de sessão de depuração de pesquisa**.
3. Se ainda não inscrito com o Application Insights, a janela pesquisa será aberto em modo de telemetria de sessão de depuração.
4. Clique em de **pesquisa** ícone para ver a telemetria local.

![Carregar concluída](./media/app-insights-release-notes-vsix/LocalSearch.png)

Se já tiver uma destas arquiteturas de registo configuradas, a seguinte secção descreve como a obter.
**Se já adicionou o Application Insights:**

1. Clique no nó do projeto e, em seguida, clique em **Application Insights**e, em seguida, clique em **configurar o Application Insights**. Certifique-se de que vê a opção para adicionar o adaptador correto na janela de configuração.
2. Em alternativa, ao criar a solução, tenha em atenção a janela de pop-up que aparece na parte superior direita do ecrã e clique em **configurar**.

### <a name="see-all-telemetry-events-associated-with-the-event"></a>Ver todos os eventos de telemetria associados com o evento
Adicionámos um novo separador, com consultas do predefinidas para todos os dados relacionados com eventos de telemetria que o utilizador está a visualizar, junto ao separador de detalhes do evento. Por exemplo, um pedido tem um campo chamado **ID da operação**. Todos os eventos associados a este pedido tem o mesmo valor para **ID da operação**. Se ocorrer uma excepção durante a operação está a processar o pedido, a exceção é atribuída o mesmo ID de operação que o pedido para tornar mais fácil localizar. Se estiver à procura a um pedido, clique em **toda a telemetria para esta operação** para abrir um novo separador que apresenta os resultados da pesquisa de novo.

### <a name="jump-from-an-exception-to-method-in-source-code"></a>Ir a partir de uma exceção ao método no código fonte

Só tem de ter o projeto correto carregado e o Application Insights encarrega-se de que o resto! (Para obter mais informações sobre a janela de pesquisa do Application Insights, consulte as notas de versão para versão 4.0 nas seguintes secções.)

Como funciona? Pode utilizar a procura de informações de aplicações, mesmo quando uma solução não está aberta. A área de rastreio de pilha apresenta uma mensagem de informações e muitos dos itens no rastreio de pilha não estão disponíveis.

Se as informações do ficheiro estão disponíveis, alguns itens podem ser ligações, mas o item de informações de solução ainda vai estar visível.

Se clicar na hiperligação, terá de ir para a localização do método selecionada no seu código. Pode existir uma diferença no número de versão, mas a funcionalidade, para ir para a versão correta do código, ficará em versões posteriores.
